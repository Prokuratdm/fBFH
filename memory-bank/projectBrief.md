# Project Brief: fBFH

**Полное название**: front for bFH (Big Friendly Hockey)

**Платформа**: Flutter Web (основная), с поддержкой iOS и Android в будущем.

**Цель**: Frontend-приложение для системы управления хоккейными клубами: пользователи, команды, дети, тренировки, нормативы, инвентарь.

**Бэкенд**: REST API на `localhost:8080`, JWT-аутентификация.

**Ключевые требования:**
- Логин/пароль → JWT-токен, сохранение в куках браузера
- Автоматическая валидация токена при перезапуске через `/api/v1/auth/me`
- Динамическое меню на основе ролей пользователя (6 ролей)
- i18n: русский по умолчанию, английский запасной
- Браузерный Password Manager (сохранение логина/пароля)
- Сворачиваемое боковое меню с иконками
- Переменные окружения через --dart-define
