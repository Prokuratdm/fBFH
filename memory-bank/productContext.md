# Product Context: fBFH

## Зачем проект существует
Веб-интерфейс для администраторов, методистов, тренеров и представителей хоккейных клубов. Позволяет управлять тренировочным процессом: вести учёт детей, нормативов, инвентаря, строить тренировки через drag-and-drop.

## Проблемы, которые решает
- Бумажный учёт нормативов и тренировок → цифровой
- Разрозненные данные по клубам → единая система
- Отсутствие шаблонов тренировок → переиспользуемые эталонные тренировки
- Сложность планирования → календарь + конструктор тренировок

## Как должно работать
1. Пользователь вводит логин/пароль
2. Система возвращает JWT-токен и роли
3. Токен сохраняется в куках, используется для всех API-запросов
4. При перезапуске приложение проверяет токен через `/me`
5. Меню формируется динамически на основе ролей
6. Браузер предлагает сохранить пароль

## Пользовательский опыт
- Минималистичный Material Design 3
- Боковое меню с иконками, сворачивается для экономии места
- Русский интерфейс по умолчанию
- Мгновенная обратная связь при ошибках (SnackBar)
